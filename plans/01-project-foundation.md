## Step 1: Initialize the Expo Project

> **Important:** `@react-native-firebase` requires native modules that are not available in Expo Go. This project uses **Continuous Native Generation (CNG)** — `ios/` and `android/` directories are generated by `expo prebuild` and should not be manually edited. Do NOT use the `bare-minimum` template; use the default template with CNG.

> **SDK Version:** The project uses **Expo SDK 54** with **React Native 0.81.5** and **expo-router ~6.0.23**. NativeWind v5 requires RN 0.81+ and the New Architecture.

1. Run `npx create-expo-app@latest triptrack` (default template — TypeScript, expo-router pre-configured).
   If the target directory already contains files, scaffold in a temporary directory and copy the generated files (assets, package.json, tsconfig.json, etc.) into the project.
2. Update `tsconfig.json` to add the path aliases used throughout the project:
   ```json
   {
     "extends": "expo/tsconfig.base",
     "compilerOptions": {
       "strict": true,
       "paths": {
         "@/*": ["./src/*"],
         "@/tw": ["./src/tw/index"],
         "@/tw/image": ["./src/tw/image"],
         "@/tw/animated": ["./src/tw/animated"]
       }
     },
     "include": [
       "**/*.ts",
       "**/*.tsx",
       ".expo/types/**/*.ts",
       "expo-env.d.ts"
     ]
   }
   ```
3. Generate the native projects:
   ```bash
   npx expo prebuild
   ```
   This creates `ios/` and `android/` from the config plugins. Re-run after adding any new native package.
4. Verify the project builds:
   ```bash
   npx expo run:ios   # requires Xcode
   npx expo run:android  # requires Android Studio
   ```
   **Do NOT use `npx expo start` with Expo Go** — `@react-native-firebase` and ML Kit require a custom build.

## Step 3: Configure NativeWind v5 / Tailwind CSS v4

NativeWind v5 uses **Tailwind CSS v4** and the **react-native-css** runtime. The key differences from v4:
- No `tailwind.config.ts` — theme tokens are defined in CSS `@theme` blocks inside `global.css`
- No babel preset for NativeWind — Metro handles it via `withNativewind`
- All React Native core components (`View`, `Text`, etc.) must be wrapped with `styled()` from `react-native-css` to accept `className`. These wrappers live in `src/tw/` (Step 6.5)
- `babel.config.js` is not needed — Expo SDK 54 does not generate one by default

> **Critical API note:** `react-native-css` v3 exports both `styled()` (HOC factory) and `useCssElement()` (React hook). The `styled()` function can be called at module level to create wrapped components. `useCssElement()` is a React hook that uses `useState` internally and **cannot** be called at module level — it must be called inside a React component. Use `styled()` for `src/tw/` wrapper exports.

### 1. `metro.config.js`

> **NativeWind v5 change:** The `input` option has been removed from `withNativewind` in v5. CSS-first configuration is now handled entirely in `global.css` via `@import` directives. Just call `withNativewind(config)` with no second argument.

```js
const { getDefaultConfig } = require("expo/metro-config");
const { withNativewind } = require("nativewind/metro");

const config = getDefaultConfig(__dirname);

module.exports = withNativewind(config);
```

### 2. `postcss.config.mjs`

```js
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

### 3. `global.css`

```css
@import "tailwindcss/theme.css" layer(theme);
@import "tailwindcss/utilities.css" layer(utilities);
@import "nativewind/theme";

/* All Material Design 3 tokens and Lexend font are defined here — see Step 4 */
```

> **Important:** The `@import "nativewind/theme"` directive is required for NativeWind v5 to register its native CSS functions (`platformColor()`, `hairlineWidth()`, etc.). Without it, NativeWind theme features will not work correctly.

### 4. `babel.config.js`

Expo SDK 54 does not generate a `babel.config.js` by default — the Expo preset is applied automatically by Metro. If a `babel.config.js` exists and only contains `{ presets: ['babel-preset-expo'] }`, delete it. Do NOT add a NativeWind babel preset (v5 does not use one).

### 5. `nativewind-env.d.ts` — TypeScript type augmentation

Create a `nativewind-env.d.ts` file at the project root to enable TypeScript to recognize the `className` prop on React Native components:

```typescript
/// <reference types="nativewind/types" />
```

This references the type augmentation from `react-native-css/types.d.ts` (via `nativewind/types.d.ts`), which adds `className?: string` to `ViewProps`, `TextProps`, `PressableProps`, etc.

### 6. Root layout import

In `app/_layout.tsx`, import `global.css` and the `expo-sqlite` localStorage polyfill at the top:
```typescript
import "../global.css";
import "expo-sqlite/localStorage/install"; // global localStorage backed by SQLite (used by driveService queue — Plan 10)
```

## Step 6.5: Create `src/tw/` CSS-Wrapped Component Exports

> **Critical:** NativeWind v5 requires all React Native core components to be wrapped with `styled()` from `react-native-css` before they can accept a `className` prop. Raw `react-native` components will silently ignore `className`. All code in this project **must** import `View`, `Text`, `ScrollView`, `Pressable`, `TextInput`, and `Image` from `@/tw` (not from `react-native` or `react-native-reanimated`).

> **API choice:** Use `styled()` (HOC factory), NOT `useCssElement()` (React hook). `styled()` can be called at module level to produce a wrapped component. `useCssElement()` uses `useState` internally and can only be called inside a React component body — it is not suitable for creating reusable wrapper exports.

### `src/tw/index.tsx`

```typescript
import {
  View as RNView,
  Text as RNText,
  ScrollView as RNScrollView,
  Pressable as RNPressable,
  TextInput as RNTextInput,
  TouchableHighlight as RNTouchableHighlight,
} from "react-native";
import Animated from "react-native-reanimated";
import { Link as ExpoLink } from "expo-router";
import { styled } from "react-native-css";

export const View = styled(RNView);
export const Text = styled(RNText);
export const ScrollView = styled(RNScrollView);
export const Pressable = styled(RNPressable);
export const TextInput = styled(RNTextInput);
export const TouchableHighlight = styled(RNTouchableHighlight);
export const AnimatedView = styled(Animated.View);
export const AnimatedScrollView = styled(Animated.ScrollView);
// eslint-disable-next-line @typescript-eslint/no-explicit-any
export const Link = styled(ExpoLink as any);
```

### `src/tw/image.tsx`

```typescript
import { Image as ExpoImage } from "expo-image";
import { styled } from "react-native-css";

export const Image = styled(ExpoImage);
```

### `src/tw/animated.tsx`

```typescript
import Animated from "react-native-reanimated";
import { styled } from "react-native-css";

export const AnimatedView = styled(Animated.View);
export const AnimatedText = styled(Animated.Text);
export const AnimatedScrollView = styled(Animated.ScrollView);
```

### Usage across the codebase


## Step 7: Define TypeScript Types

> **Two-layer type system:** This project has two type layers that serve different purposes:
> - `src/types/index.ts` (this file) — **UI view models**. Lightweight interfaces shaped for component props, store state, and local computations. May omit Firestore-specific fields like `syncStatus` or `_pendingWrite`.
> - `src/types/firestore.ts` — **Firestore document shapes** (defined in Plan 09). These are the canonical data structures with all `userId`, `tripId`, `syncStatus`, timestamps, and server-side fields. Use Firestore types whenever reading/writing to Firebase.
>
> Services and stores bridge between the two layers: services accept/return Firestore types, stores expose UI types derived from Firestore types.

Create `src/types/index.ts`:

```typescript
import auth from "@react-native-firebase/auth";

/**
 * Returns the current user's UID or throws if unauthenticated.
 * Use this instead of auth().currentUser!.uid to get a clear error.
 */
export function requireAuth(): string {
  const uid = auth().currentUser?.uid;
  if (!uid) throw new Error("requireAuth: no authenticated user");
  return uid;
}
```
